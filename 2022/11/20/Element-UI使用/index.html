Element UI使用

发表于2022-11-20|更新于2023-07-30
|阅读量:

Table 表格

:data绑定数据源 table中可以嵌套很多模板列el-table-column

label指定当前列的标题 prop指定当前列指向的数据

Switch 开关

通过作用域插槽 template 来渲染了状态列，

通过slot-scope接收了当前作用域的数据，

通过 scope.row拿到这一行的数据，

通过v-model绑定具体的属性值把开关的状态绑定到每个属性身上

如果同时指定了prop和作用域插槽，那么作用域插槽会覆盖prop，prop不再生效可删除

通过包裹tooltip进行消息提示

:enterable=”false” 鼠标进入tooltip消息自动隐藏

Pagination分页组件

定义size-change、current-change事件

在事件中接受到最新的pagesize和页码值

只要pagesize和页码值改变，就立即发起一次新的数据请求

Input框

clearable是×清空搜索框内容 绑定clear事件清空搜索的内容显示原本内容

Dialog对话框的展示

按需导入对话框组件 提供title标题 span为内容主体 下面的span为底部区域包括确定和取消按钮

:visible.sync控制显示与隐藏

为button绑定addDialogVisible = false隐藏

点击添加用户为button绑定事件,将addDialogVisible设为true

<el-button type="primary" @click="addDialogVisible = true">添加用户</el-button>

带验证规则的Form表单

添加model数据绑定对象，提供rules验证规则 ref为引用对象 label-width指定了前面文本所占的宽度

通过label来指定前面的文本 prop来指定具体的校验规则，校验规则一定要在addFormRules里面添加

在每个el-form-item里面会包含一个文本输入框 通过v-model来双向绑定，绑定到addForm数据域，以后输入的数据都会自动同步到addForm中

MessageBox

需全局引入

通过原型挂载的形式将 MessageBox.confirm这个函数挂载到vue的原型对象$confirm上

Vue.prototype.$confirm = MessageBox.confirm

在每个组件中都可以通过this.$confirm来弹出一个确认消息的提示框，在提示框中通过.catch捕获了用户的取消行为，当用户无论点击了确认还是取消，都可以用async await来简化这次promise操作，通过confirmResult接收到这次操作结果，拿到confirmResult来进行这次if else的判断

El-Table使用

stripe隔行变色
